Jmeter Thread Group - Sun, Mar 11, 2018

1、Action to be take after a Sampler error

  • Continue：忽略错误，继续执行操作
  • Start Next Thread Loop：忽略错误，线程当前循环终止，执行下一个循环
  • Stop Thread：停止当前线程，其他线程不受影响
  • Stop Test：当前执行的采样器结束后，停止整个测试计划
  • Stop Test Now：立即停止整个测试计划

2、Thread Properties

  • Number of Threads(users)：当前线程数量，可以简单的理解为用户数量。
  • Ramp-up Period (in seconds)：达到上面指定线程数所花费的时间，单位为秒。举个栗子：假设线程数为100个，花费时间20s，那么每秒启动的线程数 = 线程数/时间，即100/20 = 5。换句话说，就是1秒启动5个线程。
  • Loop Count：勾选“永远”选项，则线程组一直循环。否则，以后面所填数量为准。
  • Delay Thread creation until needed：当线程需要执行的时候，才会被创建。如果不勾选此选项，所有线程在开始时就全部被创建。
  • Scheduler：勾选此选项，才可修改下面的调度器配置

3、Scheduler Configuration

  • Duration(seconds)：整个测试计划持续的时间。（优先于后面的结束时间选项)
  • Startup delay(seconds)：测试计划启动后，会被延迟启动，时间为选项填入的时间。（优先于后面的启动时间选项）
